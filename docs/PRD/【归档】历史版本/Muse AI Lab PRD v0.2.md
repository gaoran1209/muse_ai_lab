## 0. 一句话定义

`MUSE AI Lab` 是面向服装与时尚行业的、连接内容创作者和达人的双端 Web 应用：

*   `Muse Spark`：给内容创作者使用的"创作工作台"，用于选款、搭配、AI 生成图/视频、管理内容产出。
    
*   `Musey Land`：给达人使用的"沉浸式内容场"，用于浏览、互动、种草、反馈偏好。
    

核心目标是打通"素材 -> 搭配 -> 生成 -> 发布 -> 互动反馈 -> 反哺下一轮创作"的闭环。

## 1. 背景与问题

### 1.1 现状痛点

*   服装内容生产链路割裂：灵感、选款、搭配、拍摄、发布、反馈分散在多个工具中。
    
*   高质量内容成本高、周期长：拍摄组织重、复用低、试错慢。
    
*   消费端反馈回流慢：达人偏好无法快速反哺到创作端。
    

### 1.2 机会点

*   用 Demo 证明：
    
    *   通过 AI 把"搭配 + 视觉生成 + 反馈分析"串成统一工作流。
        
    *   少人团队借助 AI coding 也能快速产出可演示、可迭代的时尚内容系统。
        

### 1.3 竞品定位与差异化

#### 1.3.1 竞品格局概览

市面上的 AI 时尚工具可分为四类，但没有任何一款打通"搭配创作 → AI 生成 → 消费者互动 → 数据回流"的完整链路：

| 竞品类别 | 代表产品 | 核心能力 | 缺失环节 |
| --- | --- | --- | --- |
| **通用 AI 创意画布** | TapNow | 多模型聚合生成（VEO 3.1/Kling/Midjourney）、节点式工作流、电商全链路出图 | 无搭配逻辑、无消费端、无反馈闭环 |
| **AI 电商摄影** | Claid、SellerPic、Botika、Photoroom | 平铺图→模特图、AI 换背景、批量电商图 | 只做拍摄环节，无上游搭配、无下游分发 |
| **AI 时尚设计** | Style3D、The New Black、Refabric | 草图→3D 建模、面料模拟、设计迭代 | 偏重设计研发，不做内容生产和消费端 |
| **虚拟试穿** | FASHN AI、WearView、FitRoom | 消费者端虚拟换装、体型适配 | 无创作工作台，无 B 端工具 |

#### 1.3.2 核心竞品深度分析：TapNow

TapNow（Tamar AI 旗下）是目前最接近的对标产品，定位为"AI 原生创意画布"，月访问量约 72 万（中国市场占 63%）。

**TapNow 的优势**：

*   模型聚合深度：一个平台接入 VEO 3.1、Kling、Midjourney、Imagen 等多家生成模型
    
*   Tapflow 节点式工作流：给高阶用户提供灵活的创意编排能力
    
*   电商全链路：上传 3 张产品图 + 2 个场景 → 自动生成主图/模特图/详情页，支持多平台适配
    
*   社区飞轮：TapTV 创作者社区形成内容共享和二创文化
    
*   商业化成熟：积分制（Tapies），Starter $90/年 ～ Enterprise $2160/年
    

**TapNow 的短板（= MUSE 的机会）**：

*   不理解服装搭配：缺乏品类搭配规则、风格一致性、季节适配等时尚行业知识
    
*   无消费端闭环：只有创作者社区，没有面向达人/消费者的沉浸式内容场
    
*   无反馈回流机制：内容生成后链路中断，无法用消费端偏好信号反哺下一轮创作
    
*   节点式学习曲线高：对非技术背景的设计师/买手来说，节点编排门槛较高
    

#### 1.3.3 MUSE AI Lab 的差异化定位

**一句话定位**：MUSE AI Lab 不只帮你生成内容，更帮你验证哪些内容消费者真正买单。

**两个核心壁垒**：

1.  **搭配智能**：内置服装行业的搭配知识图谱（品类关系、风格一致性、季节适配、场合匹配），AI 不只是生成图片，而是理解"什么和什么搭在一起好看"。
    
2.  **消费端闭环**：Muse'y 让内容直接触达达人，互动数据（点赞、收藏、评论、选品偏好）实时回流 Muse Spark，驱动下一轮创作决策——这是纯生成工具做不到的。
    

## 2. 产品目标（Demo 阶段）

### 2.2 产品目标（可验证）

*   创作者可从本地导入图片到 `Muse Spark` 中，完成选品和搭配
    
*   每套搭配可触发 AI 虚拟拍摄（图像为主，视频可为简化版）并支持反复修改，最终采纳进入结果流。
    
*   生成的内容可发布到 `Musey Land`，并支持点赞/收藏/评论/选款等互动。
    
*   达人可以在 `Musey Land` 中提交自己的 TryOn 任务，展示自己的试穿效果。图文内容即刻展示结果；视频任务分发到 Muse Spark 中，供创作者领取——创作——发布
    

### 2.3 非目标（本期不做）

*   不做完整电商交易链路（下单/支付/履约）。
    
*   不做复杂组织权限体系（先用简化角色）。
    
*   不做大规模高并发和全球化架构（以 Demo 稳定性为先）。

## 4. 用户旅程（User jounery）

**前端分Muse Spark和Muse Land两部分**

**Muse Spark创作流程（面向：内容创作者，设计师/买手/运营）**

1.  新建设计方案，导入或选择素材（自动打标 + 人工修正）。
    
2.  在画布中选中 1～2 件单品，触发 AI 搭配补全，根据选中的单品生成多组 Look 到画布 Board 中。
    
    1.  或者：选择多个单品，触发 AI 搭配分组，直接生成多组 look
        
3.  浏览对比 Look 方案，替换/调整单品，满意后确认 Look。
    
4.  选中 Look，点击"拍摄"，配置模特/场景/效果 后生成效果图。
    
5.  查看生成结果，不满意则调整参数重新生成；满意则点击"采纳"进入结果面板，也可以根据效果图生成视频
    
6.  从结果面板选择已采纳内容，发布至 Musey Land。
    

**Muse Land 消费流程（达人）**

1.  进入 Musey Land 浏览内容流。
    
2.  对内容点赞/收藏/评论。
    
3.  打开内容详情查看图文详情、相关商品
    
4.  对喜欢的内容，发起 TryOn (参考图片风格、商品、场景，生成自己形象的图文或视频)
    
    1.  图文：调用生图模型，直接展示TryOn 结果
        
    2.  视频：生成任务，给 MuseSpark 工作台，让创作者领取任务、创作内容、交付
        

本期 Demo 版本的范围严格围绕“创作者内容生产”与“达人消费互动”的双端动线展开。我们将产品模块按用户的实际操作流程切分为两大部分：

### 4.1 Muse Spark

**（面向群体：内容创作者、设计师、买手、运营）**

#### 4.1.1 创建画布

*   **Dashboard管理：** 用户登录后查看和管理历史的设计方案，或新建方案开启一张全新的空白画布。
    
*   **素材导入与处理：** 用户从本地批量导入单品图、模特图等素材。系统自动为素材打标分类，用户可进行人工修正确认。
    

#### 4.1.2 搭配成组

*   **画布自由组织：** 用户将素材拖拽进入画布，可自由摆放图片、添加文本，或将多个元素框选为组合。
    
*   **AI 搭配补全：** 用户在画布中选中 1～2 件核心单品并触发，系统自动生成 3～5 组完整的搭配方案（包含补齐的单品及风格描述），呈现在画布的独立看板中。
    
*   **AI 搭配分组：** 用户选中 3 件以上单品并触发，系统自动对选品进行逻辑排列组合，生成多套搭配方案。
    
*   **人工微调：** 用户在生成的看板中进行比对，可直接拖拽替换不满意的单品，确认最终的搭配组合。
    

#### 4.1.3 虚拟拍摄

*   **配置拍摄参数：** 选中确认好的搭配组合，点击“拍摄”。在配置面板中选择预设的模特、场景、姿势、效果，并支持输入自定义文本提示词补充细节。
    
*   **任务调度与预览：** 提交任务后，后端进行排队与生成处理。生成的结果图直接展示在画布中对应搭配的旁边。用户如果不满意可调整参数重新生成。
    
*   **视频生成：** 支持文生视频或图生视频，配置面板中选择视频生成，即可出现视频节点，支持输入自定义文本提示词补充细节生视频
    

#### 4.1.4 效果定稿

*   **结果采纳：** 用户对满意的生成结果点击“采纳”，该图片及其绑定的搭配单品明细、参数配置将打包进入右侧的结果面板。（图片仍保留在画布中供排版编辑）。
    
*   **发布上架：** 用户在结果面板中选中已采纳的内容，点击发布，将其推送至 Muse Land 的内容前台。
    

### 4.2 Muse Land

**（面向群体：达人、消费者）**

#### 4.2.1 内容浏览与互动

*   **内容流浏览：** 达人进入前台，通过瀑布流或卡片流浏览最新上架的图文/视频内容，支持按合集或推荐逻辑查看。
    
*   **内容详情与互动：** 对感兴趣的内容进行点赞、收藏、评论。进入详情页可查看大图、搭配单品明细及风格文案。
    
*   **商品流转：** 针对内容中包含的具体单品，提供生成带货链接的入口。
    

#### 4.2.2 发起 TryOn 试穿

*   **达人基于喜欢的参考内容**（包含其风格、商品、场景），发起虚拟试穿请求，生成包含自己形象的图文或视频：
    
*   **图文 TryOn（实时响应）：** 达人上传个人照片，系统直接调用该内容背后的生图模型与工作流配置，即刻在线展示试穿结果。
    
*   **视频 TryOn（任务接单）：** 达人发起视频试穿请求，系统生成任务订单并派发至 Muse Spark 工作台。创作者在线领取任务、完成视频创作后，交付给对应达人。
    

### 全流程

#### 创作与打包 (Muse Spark 端) 

*   **交互动作：** 创作者上传 1 张核心单品图 -> 拖入画布 -> 选中后点击“补全搭配” -> 系统在画布上自动补齐其他服饰形成一个完整的 Look (Group) -> 点击“拍摄” -> 选择模特/背景/人脸—>调用图像生成 API 产出上身效果图。
    
*   **必通标准：** 生成的结果图必须能一键“采纳”，并且在后台成功绑定该 Look 包含的“所有图片信息”形成一个完整的数据包。
    

#### 内容分发 (Spark ➜ Land) 

*   **交互动作：** 创作者在右侧结果面板中，对已采纳的图片点击“发布到 Musey Land”。
    
*   **必通标准：** 数据无缝写入数据库。刷新 Musey Land 的前台页面（Feed 流），该图立刻出现，且用户点开详情能完整看到对应的“搭配单品列表”。（Demo阶段无需审核）
    

#### 工作流调用 (Land ➜ Spark) 

*   **交互动作：** 达人/KOL 账号在 Musey Land 刷到该 Look。不做复杂的评论转发，核心只保留一个强交互：点击\*\*“发起 TryOn（试穿请求）”\*\*。
    
*   **必通标准：** 达人点击 TryOn，弹窗上传一张自己的正面照（或选择预设头像）并提交。在线拿到结果
    

#### 任务接单互通 (Land ➜ Spark) 

*   **交互动作：** 视频任务采取任务单单形式传递，创作者切回 Muse Spark。在界面显著位置（如“任务中心”或画布旁的“互动看板”）收到信息提示接单
    
*   **必通标准：** 创作者点击该任务，画布自动将 Look A 的单品、达人 X 的照片以及之前制作该视频时的工作流拉入工作区，完成后，能将成片一键返还给达人 X，完成最终业务闭环。
    

## 5.产品设计

### 5.1 Muse Spark（时尚版 TapNow）

Muse Spark 端核心形态是"画布工作台"，即" 左侧素材库 + 中部画布操作区 + 右侧采纳结果列表"。

**P0 需求 list（Demo 必做）**

1.  Dashboard：
    
    *   展示方案 list（以设计方案为单位，每个设计方案中可以包含很多套搭配、很多套内容）
        
2.  `素材库`：上传/导入素材，支持批量多图片导入
    
3.  `画布`：核心操作和交互空间，包括选品、拖拽搭配、看组合、替换单品、AI拍摄、视频生成等。
    

*   `对象节点`：画布支持 3 种对象节点：文本、图片、视频。
    
*   `组合`：将画布中对象组合，形成组合（group）。
    
*   `AI 搭配`：基于 LLM 的智能搭配推荐，包含搭配补全和搭配分组。
    
*   `拍摄`：基于选定的图片，生成拍摄任务，拍摄按钮展开生成面板的功能，点击后展开
    
    *   换模特
        
    *   换背景
        
    *   换姿势
        
    *   换效果
        
    *   换服装
        
    *   自定义指令
        

1.  `结果面板`：展示采纳结果（支持图片和视频）。
    

*   `发布`按钮：可将结果发送到 Musey Land。
    

#### 5.1.2 AI 搭配建议

AI 搭配建议是 MUSE 区别于通用 AI 创意工具的核心功能。

##### 4.1.2.1 交互模式

| 输入场景 | 用户操作 | 系统行为 | 输出 |
| --- | --- | --- | --- |
| **搭配补全** | 选中 1～2 件单品 | LLM 基于标签 + 搭配规则，生成 3～5 组完整 Look（每个 look 包含多个商品图） | 每组 Look 直接放入画布的独立 Board 区域，附带风格文案描述 |
| **搭配分组** | 选中 3+ 件单品 | LLM 对选品进行排列组合，生成多个 Look 方案 | 每个组合占据一个 Board 区域，用户可快速对比筛选 |

##### 5.1.2.2 搭配逻辑（LLM Prompt 驱动）

MVP 阶段采用 LLM prompt 工程实现搭配智能，核心 prompt 结构详见 [llm-prompt.md](llm-prompt.md)

##### 5.1.2.3 输出形式

*   搭配结果直接渲染到画布中，每组 Look 占据一个独立的 Board 区域
    
*   每个 Board 包含：单品缩略图排列 + 搭配文案描述 + 风格标签
    
*   用户可在 Board 中替换单品（拖拽新单品替换）、调整后重新触发推荐
    
*   MVP 阶段进阶能力：选中一件主品 → LLM 推荐搭配组合 + 调用图像生成模型生成对应其他商品的效果图
    

#### 5.1.3 拍摄生成

##### 5.1.3.1 触发方式

在搭配画布中选中一组 Look 后，点击"拍摄"按钮展开生成面板。

##### 5.1.3.2 生成面板（可控项）

| 控制项 | 交互形式 | 说明 |
| --- | --- | --- |
| **模特选择** | 预设模板库（不同肤色/体型/性别） | MVP 阶段提供 6～8 个预设模特选项 |
| **场景选择** | 预设场景模板（街拍/棚拍/户外/室内） | MVP 阶段提供 8～10 个预设场景 |
| **姿势选择** | 预设姿势参考（可选） | 站姿/坐姿/行走等基础姿势 |
| **效果选择** | 预设效果模板（可选） | 不同场景下的特殊效果（如时尚大片风格/高级摄影棚风格/灰调美丽写真等） |
| **自定义指令** | 自由文本 prompt 输入框 | 用户可输入额外风格/氛围描述，叠加到生成 prompt |

预设模板和自由 prompt 可组合使用：用户选一个预设场景后，仍可通过文本补充细节描述。

##### 视频策略

Demo 阶段**仅做图文生**视频这一场景。

##### 5.1.3.3 生成任务流程

```plaintext
用户点击"拍摄" → 创建 GenerationTask（状态：排队）
    → 后端调度生成（状态：处理中）
    → 生成完成，返回结果图（状态：完成）/ 失败重试（状态：失败）

```

*   每次拍摄生成 **1 张** 结果图，输出直接展示在画布中
    
*   用户不满意可调整参数后重新生成（生成新任务，不覆盖旧结果）
    
*   用户在画布中可查看本次 Look 的所有历史生成结果
    

##### 5.1.3.4 "采纳"操作定义

*   用户对画布中的某张生成图点击"采纳"按钮
    
*   采纳后：该图进入右侧**结果面板**，标记为"已采纳"状态
    
*   采纳后图片**同时保留在画布中**，用户可继续编辑（添加标注、调整排版等）
    
*   结果面板中的已采纳图片可以执行"发布到 Musey Land"操作
    
*   用户可取消采纳（从结果面板移除），不影响画布中的图片
