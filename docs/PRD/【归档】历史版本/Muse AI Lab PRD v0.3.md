## 0. 一句话定义

MUSE AI Lab 是面向时尚行业的 **AI 内容创作与验证平台**：创作者通过 AI 快速生成搭配内容（Muse Spark），达人通过互动和试穿验证内容价值（Muse Land），互动数据实时回流驱动下一轮创作。

*   **Muse Spark**：给内容创作者使用的"创作工作台"——选款、AI 搭配、虚拟拍摄、内容管理、数据回看。

*   **Muse Land**：给达人使用的"沉浸式内容场"——浏览、互动、试穿、种草、反馈偏好。

## 1. 背景与问题

### 1.1 现状痛点

*   服装内容生产链路割裂：灵感、选款、搭配、拍摄、发布、反馈分散在多个工具中。

*   高质量内容成本高、周期长：拍摄组织重、复用低、试错慢。

*   消费端反馈回流慢：达人偏好无法快速反哺到创作端。

### 1.2 机会点

*   用 Demo 证明：

    *   通过 AI 把"搭配 + 视觉生成 + 反馈分析"串成统一工作流。

    *   少人团队借助 AI coding 也能快速产出可演示、可迭代的时尚内容系统。

### 1.3 竞品定位与差异化

#### 1.3.1 竞品格局概览

市面上的 AI 时尚工具可分为四类，但没有任何一款打通"搭配创作 → AI 生成 → 消费者互动 → 数据回流"的完整链路：

| 竞品类别 | 代表产品 | 核心能力 | 缺失环节 |
| --- | --- | --- | --- |
| **通用 AI 创意画布** | TapNow | 多模型聚合生成（VEO 3.1/Kling/Midjourney）、节点式工作流、电商全链路出图 | 无搭配逻辑、无消费端、无反馈闭环 |
| **AI 电商摄影** | Claid、SellerPic、Botika、Photoroom | 平铺图→模特图、AI 换背景、批量电商图 | 只做拍摄环节，无上游搭配、无下游分发 |
| **AI 时尚设计** | Style3D、The New Black、Refabric | 草图→3D 建模、面料模拟、设计迭代 | 偏重设计研发，不做内容生产和消费端 |
| **虚拟试穿** | FASHN AI、WearView、FitRoom | 消费者端虚拟换装、体型适配 | 无创作工作台，无 B 端工具 |

#### 1.3.2 核心竞品深度分析：TapNow

TapNow（Tamar AI 旗下）是目前最接近的对标产品，定位为"AI 原生创意画布"，月访问量约 72 万（中国市场占 63%）。

**TapNow 的优势**：

*   模型聚合深度：一个平台接入 VEO 3.1、Kling、Midjourney、Imagen 等多家生成模型

*   Tapflow 节点式工作流：给高阶用户提供灵活的创意编排能力

*   电商全链路：上传 3 张产品图 + 2 个场景 → 自动生成主图/模特图/详情页，支持多平台适配

*   社区飞轮：TapTV 创作者社区形成内容共享和二创文化

*   商业化成熟：积分制（Tapies），Starter $90/年 ～ Enterprise $2160/年

**TapNow 的短板（= MUSE 的机会）**：

*   不理解服装搭配：缺乏品类搭配规则、风格一致性、季节适配等时尚行业知识

*   无消费端闭环：只有创作者社区，没有面向达人/消费者的沉浸式内容场

*   无反馈回流机制：内容生成后链路中断，无法用消费端偏好信号反哺下一轮创作

*   节点式学习曲线高：对非技术背景的设计师/买手来说，节点编排门槛较高

#### 1.3.3 MUSE AI Lab 的差异化定位

**一句话定位**：MUSE AI Lab 不只帮你生成内容，更帮你验证哪些内容消费者真正买单。

**两个核心壁垒**：

1.  **搭配智能**：内置服装行业的搭配知识图谱（品类关系、风格一致性、季节适配、场合匹配），AI 不只是生成图片，而是理解"什么和什么搭在一起好看"。

2.  **消费端闭环**：Muse Land 让内容直接触达达人，互动数据（点赞、收藏、TryOn 发起率、风格偏好）实时回流 Muse Spark，驱动下一轮创作决策——这是纯生成工具做不到的。

### 1.4 Demo 阶段与整体产品路线图的关系

> **背景说明**：在前期的行业研究和商业模式评估中，MUSE AI Lab 的完整愿景是「AI 视频导演 Agent」——以编排层为核心，通过多模型智能路由实现「一句话到可用商业视频」，商业模式为 SaaS + GMV 分润。

**Demo 阶段有意做了范围收窄**，从"AI 视频导演 Agent"收窄为"AI 搭配 + 生图 + 内容消费闭环"的 MVP。原因如下：

| 完整产品能力 | Demo 阶段处理方式 | 理由 |
|---|---|---|
| 多模型编排 / 智能路由 | 暂时只接入单一生图 API + 单一视频 API | Demo 阶段优先验证搭配→生成→消费的核心链路，模型路由是工程优化问题，可后续叠加 |
| 视频导演 Agent（脚本→分镜→生成→剪辑） | 仅做图生视频（单镜头），不做多镜头编排 | 视频编排链路复杂度高，1 周内无法做到可用体验 |
| GMV 分润 / 电商闭环 | 不做（Demo 不含交易） | 需要商品库、订单系统，与核心验证无关 |
| 搭配智能 + 消费端闭环 | **重点验证** | 这是 MUSE 区别于 TapNow/Lovart 的核心差异化，Demo 必须证明这条路走得通 |

**Demo 如何为后续完整产品铺路**：

1. Demo 的画布架构、Provider 模式、Task 调度系统已设计为可扩展——后续接入新模型只需实现 Provider 接口
2. 搭配智能的 LLM prompt 框架可逐步升级为 Agent + 知识图谱
3. Muse Land 的互动数据基础设施可直接复用于后续的推荐算法和 GMV 分润计算

## 2. 产品目标（Demo 阶段）

### 2.1 核心假设与验证指标

Demo 阶段最关键的任务是验证以下业务假设：

| # | 假设 | 验证方式 | 成功标准 |
|---|------|---------|---------|
| H1 | AI 搭配建议的准确率足以支撑创作者使用 | 搭配采纳率 | ≥ 40% 的 AI 推荐 Look 被创作者采纳或微调后采纳 |
| H2 | AI 生图质量满足时尚内容发布要求 | 生成→采纳→发布的漏斗转化率 | ≥ 30% 的生成图被采纳进入结果面板 |
| H3 | 达人愿意与 AI 生成内容互动 | Muse Land 的互动率 | 点赞/收藏率 ≥ 10%（基于内容曝光量） |
| H4 | TryOn 功能对达人有吸引力 | TryOn 发起率 | ≥ 5% 的内容浏览产生 TryOn 请求 |

### 2.2 产品目标（可验证）

*   创作者可从本地导入图片到 Muse Spark 中，完成选品和搭配

*   每套搭配可触发 AI 虚拟拍摄（图像为主，视频为简化版图生视频）并支持反复修改，最终采纳进入结果流

*   生成的内容可发布到 Muse Land，并支持点赞/收藏/评论等互动

*   达人可以在 Muse Land 中发起图文 TryOn，即刻查看试穿效果

*   Muse Spark Dashboard 展示已发布内容的互动数据概览，AI 搭配时注入近期互动偏好数据

### 2.3 非目标（本期不做）

*   不做完整电商交易链路（下单/支付/履约）

*   不做复杂组织权限体系（Demo 阶段只有 Spark 用户和 Land 用户两种入口，不区分买手/创作者角色）

*   不做大规模高并发和全球化架构（以 Demo 稳定性为先）

*   不做视频 TryOn 任务接单系统（延后到 Phase 1）

*   不做带货链接生成（延后到 Phase 1）

*   不做复杂推荐算法（Demo 内容量小，按时间倒序 + 简单标签筛选即可）

## 4. 用户旅程（User Journey）

> 本节只保留高层次的流程概览，具体功能定义详见第 5 节。

### 4.1 Muse Spark 创作流程

```
新建设计方案 → 导入素材（自动打标）→ 画布中选单品 → AI 搭配 → 浏览对比 Look → 确认 Look → 虚拟拍摄 → 查看/调整/重新生成 → 采纳 → 发布到 Muse Land
                                                                                                                              ↓
                                                                                                                     （可选）图生视频
```

### 4.2 Muse Land 消费流程

```
进入 Muse Land → 浏览 Feed 流 → 点赞/收藏/评论 → 打开详情（查看搭配单品）→ 发起图文 TryOn → 上传照片 → 查看试穿结果
```

### 4.3 数据回流闭环

```
Muse Land 互动数据（点赞/收藏/TryOn 发起）→ 汇总到 Spark Dashboard 数据面板 → 创作者触发 AI 搭配时，prompt 自动注入"近期高互动风格偏好" → 推荐结果隐含消费端反馈
```

## 5. 产品设计

### 5.1 Muse Spark（创作工作台）

核心形态：**左侧素材库 + 中部画布操作区 + 右侧结果面板**

#### 5.1.1 Dashboard

*   展示设计方案列表（以 Project 为单位，每个 Project 包含多个 Look 和生成内容）
*   新建方案 / 进入已有方案
*   **互动数据面板**：展示已发布内容的互动数据概览（点赞数、收藏数、TryOn 发起数、近期热门风格标签），作为反馈闭环的最简版本

#### 5.1.2 素材库

*   支持批量导入本地图片
*   系统自动打标分类（调 LLM API）：品类（上装/下装/鞋/包/配饰）、颜色、风格、季节、场合
*   用户可人工修正标签
*   素材以缩略图 + 标签列表展示，支持按品类/风格筛选

#### 5.1.3 画布

画布支持 3 种对象节点：**文本、图片、视频**。

**画布空间组织逻辑**：

| 区域 | 功能 | 视觉特征 |
|------|------|---------|
| 素材暂存区 | 从素材库拖入的原始单品，自由摆放 | 较小缩略图，无边界框 |
| Look Board | AI 生成的搭配方案，每个 Look 占一个 Board | 独立卡片/框，有明确边界线，包含单品缩略图 + 风格标签 + 搭配文案 |
| 拍摄结果区 | 每个 Look 对应的生成结果图/视频 | 紧邻对应 Look Board，有「采纳」按钮 |

基本操作：自由拖拽、缩放、框选组合（group）、文本添加。

#### 5.1.4 AI 搭配

AI 搭配建议是 MUSE 区别于通用 AI 创意工具的**核心差异化功能**。

##### 5.1.4.1 交互模式

**统一入口**：画布中选中单品后，点击「AI 搭配」按钮。系统根据选中单品数量和品类自动判断执行策略，用户无需区分"补全"或"分组"。

| 输入条件 | 系统行为 | 输出 |
| --- | --- | --- |
| 选中 1～2 件单品 | **搭配补全**：以选中单品为锚点，AI 补齐其他品类，生成 3～5 组完整 Look | 每组 Look 放入画布的独立 Board 区域 |
| 选中 3+ 件单品，且品类覆盖不完整 | **搭配补全**：AI 在已有品类基础上补齐缺失品类 | 每组 Look 放入 Board |
| 选中 3+ 件单品，且品类已足够丰富 | **搭配分组**：AI 将选中单品重新排列组合，生成多个 Look 方案 | 每个组合占据一个 Board |

##### 5.1.4.2 Look 的标准构成

一个完整 Look 包含 **3～5 件单品**，标准构成为：

*   上装（必须）
*   下装（必须）
*   鞋（必须）
*   包/配饰（可选，0～2 件）

##### 5.1.4.3 LLM 输出到画布的映射规则

1. LLM 接收选中单品的标签信息（品类、颜色、风格、季节、场合），基于搭配规则 prompt 输出推荐结果
2. 推荐结果为结构化数据：每个推荐项包含 `品类 + 颜色 + 风格 + 搭配理由`
3. 系统在用户素材库中进行**标签匹配**：按品类→风格→颜色的优先级匹配素材库中的实物图
4. 匹配到素材 → 直接将素材图放入 Board
5. 未匹配到素材 → 在 Board 中显示**占位卡片**，标注「建议补充：黑色高跟鞋」等提示，用户可自行上传补充

##### 5.1.4.4 搭配逻辑（LLM Prompt 驱动）

MVP 阶段采用 LLM prompt 工程实现搭配智能。核心 prompt 包含：

*   品类关系规则（上下装搭配、鞋包配饰协调）
*   风格一致性约束（不跨风格混搭，除非明确要求混搭风）
*   季节和场合适配
*   **近期高互动风格偏好注入**（闭环数据，来自 Dashboard 的互动数据面板）

详见 [llm-prompt.md](llm-prompt.md)

##### 5.1.4.5 输出形式

*   搭配结果直接渲染到画布中，每组 Look 占据一个独立的 Board 区域
*   每个 Board 包含：单品缩略图排列 + 搭配文案描述 + 风格标签
*   用户可在 Board 中替换单品（拖拽新单品替换）、调整后重新触发推荐
*   MVP 阶段进阶能力：选中一件主品 → LLM 推荐搭配组合 + 调用图像生成模型生成对应其他商品的效果图

#### 5.1.5 虚拟拍摄

##### 5.1.5.1 触发方式

在画布中选中一组确认好的 Look（Board），点击「拍摄」按钮展开生成面板。

##### 5.1.5.2 拍摄输入链路

> 这是 AI 生图的核心技术路径。Demo 阶段采用**方案 B**，后续可升级至方案 A。

| 方案 | 输入 | 输出 | 适用阶段 |
|------|------|------|---------|
| **方案 A**（完整虚拟试穿） | 所有单品图 + 模特参考图 → 虚拟试穿模型 | 模特穿着整套搭配的效果图 | Phase 1+ |
| **方案 B**（AI 创意图）✅ Demo 采用 | 核心单品图（1～2 张，如上装+下装）+ 其他品类文字描述 + 模特/场景参数 → 组合成 prompt | 风格一致的时尚效果图（更接近 AI 创意大片） | Demo |

**方案 B 的具体输入构成**：

1. 从 Look Board 中提取核心单品图（优先取上装和下装的实物图）
2. 其余单品转化为文字描述（如"搭配黑色尖头高跟鞋和棕色皮质手提包"）
3. 叠加用户选择的模特/场景参数和自定义 prompt
4. 最终 prompt = `核心单品图（image reference）+ 文字搭配描述 + 模特参数 + 场景参数 + 用户自定义指令`

**用户预期管理**：生成结果为"风格一致的时尚大片"，而非"精确的虚拟试穿效果"。UI 上以「AI 创意拍摄」而非「虚拟试穿」命名此功能。

##### 5.1.5.3 生成面板（控制项）

Demo 阶段精简为 **3 个核心控制项**：

| 控制项 | 交互形式 | 说明 |
| --- | --- | --- |
| **模特选择** | 预设模板库（不同肤色/体型/性别） | MVP 提供 4～6 个预设模特选项 |
| **场景选择** | 预设场景模板（包含场景+姿势+效果的组合） | MVP 提供 4～6 个预设场景，如「街拍-行走」「棚拍-站姿」「户外-休闲」「高级摄影棚-时尚大片」 |
| **自定义指令** | 自由文本 prompt 输入框 | 兜底所有高级需求（姿势微调、氛围描述、特殊效果等） |

> 注意：「换服装」不出现在拍摄面板——服装在搭配阶段确定。如需调整服装，应回到 Look Board 替换单品后重新拍摄。

##### 5.1.5.4 视频策略

Demo 阶段仅做**图生视频**：从已生成的效果图出发，调用视频生成 API 生成短视频。视频节点展示在画布中，支持自定义 prompt 补充动态描述。

##### 5.1.5.5 生成任务流程

```plaintext
用户点击"拍摄" → 创建 GenerationTask（状态：排队）
    → 后端调度生成（状态：处理中）
    → 生成完成，返回结果图（状态：完成）/ 失败重试（状态：失败）
```

*   每次拍摄生成 **1 张** 结果图，输出直接展示在画布中（紧邻对应 Look Board）
*   用户不满意可调整参数后重新生成（生成新任务，不覆盖旧结果）
*   用户在画布中可查看本次 Look 的所有历史生成结果

#### 5.1.6 效果定稿

##### 5.1.6.1 "采纳"操作定义

*   用户对画布中的某张生成图/视频点击「采纳」按钮
*   采纳后：该内容进入右侧**结果面板**，标记为"已采纳"状态
*   采纳后图片**同时保留在画布中**，用户可继续编辑
*   用户可取消采纳（从结果面板移除），不影响画布中的内容

##### 5.1.6.2 结果面板

结果面板**按 Look 分组展示**已采纳的图片和视频：

*   每个 Look 为一个折叠组，展示该 Look 下所有已采纳的 Shot
*   组内支持拖拽排序
*   每个 Shot 显示缩略图 + 采纳时间

##### 5.1.6.3 发布操作

*   用户在结果面板中选中一个或多个 Shot → 点击「发布到 Muse Land」
*   弹出发布编辑框：
    *   标题（支持 AI 自动生成，基于搭配的风格标签）
    *   描述文案（支持 AI 自动生成）
    *   标签（自动从 Look 的风格标签继承，可编辑）
*   确认发布 → 内容写入数据库 → Muse Land Feed 流实时展示
*   Demo 阶段无需审核

### 5.2 Muse Land（沉浸式内容场）

#### 5.2.1 内容浏览

*   **Feed 流**：瀑布流/卡片流展示已发布内容
*   **排序方式**（Demo 阶段）：默认按发布时间倒序，支持按风格标签筛选
*   **内容卡片**：封面图 + 标题 + 创作者名称 + 互动计数（点赞/收藏数）

**内容的最小发布单元**：一条 Content = 1～N 张图（来自同一个 Look 的多个 Shot）+ 标题 + 描述 + 标签。在 Feed 流中展示封面图，点击进入详情查看所有图片。

#### 5.2.2 内容详情与互动

*   大图浏览（支持左右翻页查看多图）
*   搭配单品明细（展示该 Look 包含的所有单品图 + 品类/风格标签）
*   风格文案描述
*   互动操作：**点赞、收藏、评论**

#### 5.2.3 图文 TryOn

**Demo 阶段只做图文 TryOn**，验证达人对试穿功能的需求强度。

**交互流程**：

1. 达人在内容详情页点击「TryOn 试穿」按钮
2. 弹窗引导上传一张个人正面照（或选择预设头像）
3. 提交后，系统调用 Try-On 模型（输入：达人照片 + 内容中的核心单品图 + 场景风格参数）
4. **异步生成**：提交后展示"生成中"状态，生成完成后通知达人查看
5. 结果展示：低分辨率快速预览 → 高质量版本异步生成后替换

**技术风险标注**：图文 TryOn 需要将达人人脸/体型 + 服装单品 + 场景风格融合，本质上是 virtual try-on + style transfer 的组合问题。Demo 阶段接受质量有瑕疵，核心目标是验证达人对此功能的需求意愿。

**降级方案**：如果 Try-On 模型效果不达标，可退化为"基于达人照片 + 搭配描述的 AI 创意图"（类似虚拟拍摄方案 B），而非精确试穿。

*   ~~带货链接 / 选品推荐~~（延后到 Phase 1）
*   ~~达人个人页面~~（延后到 Phase 1，但数据库需预留 profile 字段）

### 5.3 反馈闭环（最简版本）

这是 MUSE 的核心卖点——消费端数据回流驱动创作端决策。Demo 阶段实现最简闭环：

#### 5.3.1 回流数据

| 数据类型 | 来源 | 展示位置 |
|---------|------|---------|
| 点赞数 | Muse Land 互动 | Spark Dashboard 数据面板 |
| 收藏数 | Muse Land 互动 | Spark Dashboard 数据面板 |
| TryOn 发起数 | Muse Land TryOn | Spark Dashboard 数据面板 |
| 热门风格标签 | 基于高互动内容的标签聚合 | Spark Dashboard 数据面板 |

#### 5.3.2 回流到创作端的方式

1. **Dashboard 数据面板**：创作者登录 Spark 后，首页展示已发布内容的互动概览和近期热门风格趋势
2. **AI 搭配 prompt 注入**：创作者触发 AI 搭配时，系统自动在 prompt 中注入「近期高互动风格偏好」数据（如：近 7 天休闲运动风格的内容互动率最高），让推荐结果隐含消费端反馈
3. AI 搭配结果中标注「基于达人偏好推荐」的标记，让创作者感知到闭环价值

## 6. 功能优先级

基于 Demo 1 周的时间约束：

### P0（必做）

1. **Spark 素材库**：上传 + 自动打标（调 LLM API）

4. **Spark 采纳 + 发布**：采纳 → 结果面板（按 Look 分组） → 发布到 Land（支持编辑标题/描述/标签）

## 全流程验证标准（Demo 必通）

#### 创作与打包 (Muse Spark 端)

*   **交互动作**：创作者上传核心单品图 → 拖入画布 → 选中后点击「AI 搭配」→ 系统在画布上自动补齐其他服饰形成完整 Look（Board）→ 点击「拍摄」→ 选择模特/场景 → 调用图像生成 API 产出效果图

*   **必通标准**：生成的结果图能一键「采纳」，在结果面板中按 Look 分组展示，并绑定该 Look 的所有单品信息形成完整数据包

#### 内容分发 (Spark → Land)

*   **交互动作**：创作者在结果面板中选中已采纳图片 → 编辑标题/描述 → 点击「发布到 Muse Land」

*   **必通标准**：数据无缝写入数据库。刷新 Muse Land Feed 流，内容立即出现，点开详情能完整看到搭配单品列表（Demo 阶段无需审核）

#### TryOn 试穿 (Land 端)

*   **交互动作**：达人在 Muse Land 浏览到内容 → 点击「TryOn 试穿」→ 上传正面照 → 提交

*   **必通标准**：系统异步生成试穿效果图，达人能在页面中查看结果
